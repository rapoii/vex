# Java Coding Style

> Extends `rules/common/coding-style.md` with java guidance.

## Requirements

- Use clear layered boundaries, immutable value objects, constructor injection, explicit transactions, and precise exceptions.
- Keep modules cohesive and testable.
- Prefer explicit names over comments.
- Validate data entering from users, files, network, or environment.
