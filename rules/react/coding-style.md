# React Coding Style

> Extends `rules/common/coding-style.md` with react guidance.

## Requirements

- Use pure components, stable hooks, accessible semantics, server/client state separation, and no derived duplicate state.
- Keep modules cohesive and testable.
- Prefer explicit names over comments.
- Validate data entering from users, files, network, or environment.
