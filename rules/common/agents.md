# Agent Orchestration

VEX baseline rule. Apply unless a language-specific rule overrides it.

## Requirements

- Use planner for complex implementation plans.
- Use tdd-guide for bug fixes and features.
- Use code-reviewer after modifying code.
- Use security-reviewer for auth, user input, data, secrets, files, external APIs, crypto, or payments.

## Enforcement

- Prefer repo-owned tooling.
- Keep changes minimal and reversible.
- State validation evidence before calling work complete.
