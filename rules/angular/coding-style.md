# Angular Coding Style

> Extends `rules/common/coding-style.md` with angular guidance.

## Requirements

- Use standalone components, signals where useful, typed reactive forms, DI tokens, and router guards/resolvers carefully.
- Keep modules cohesive and testable.
- Prefer explicit names over comments.
- Validate data entering from users, files, network, or environment.
