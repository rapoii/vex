# Golang Coding Style

> Extends `rules/common/coding-style.md` with golang guidance.

## Requirements

- Use small interfaces at consumers, explicit errors, context propagation, table tests, and idiomatic simple structs.
- Keep modules cohesive and testable.
- Prefer explicit names over comments.
- Validate data entering from users, files, network, or environment.
