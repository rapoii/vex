# Python Coding Style

> Extends `rules/common/coding-style.md` with python guidance.

## Requirements

- Use type hints, dataclasses or Pydantic at boundaries, context managers, explicit exceptions, and structured logging.
- Keep modules cohesive and testable.
- Prefer explicit names over comments.
- Validate data entering from users, files, network, or environment.
